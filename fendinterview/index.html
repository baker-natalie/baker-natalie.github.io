  3. Describe any front-end web application framework (preferably one that you use). How does it work? What are the upsides and downsides?

    Bootstrap is the first framework I have used, and it is easy to see why it's so popular! It is simple to build functional mobile responsive pages by setting special classes to change the way the page renders based on screen size. Bootstrap's column classes also make styling the page simpler and more intuitive, since they are built on a grid. You can set the width for elements to render differently depending on the size of the screen. On a mobile phone, an image can be full width, with content elements above and below it. The same elements can be set to half-width on a desktop monitor and display side by side with the image. Other special classes allow developers to style the page as well by simply adding them to the targeted element. A minor drawback is that if the user hasn't already visited a page that utilizes Bootstrap, they may have to download it which can slow down the initial page load, but since it is such a popular framework, it's likely that most users will have the information already cached from another site.

  4. Write a JavaScript function that takes only one argument--another function--and returns a "memoized" version of that function. A "memoized" version of a function caches and returns the results of its call so that when it is called again with the same input, it doesn't run its computation but instead returns the results from cache. Note that previous results should be retrievable in any order without re-computation.

    This is a memoized function that checks a name against a list of names: If it is on the list, the function will alert the user. If the name is not on the list, it will be added. Check the console to see live results and pass your own name as a value to the namesList!
    To try it out, open the console by typing ctrl + shift + j Just type namesList('YOUR NAME HERE'); into the console and press enter! 

    
// Our memoizing function
var namesList = (function() {
  var cache = {};
  var disallowed = 'Jonathan';

  function n(name) {
    // Check to see if name is already a stored value
    if(name in cache) {
      // Alert the user that the function has pulled stored data
      console.log('cached data');
      console.log(cache);
      return cache[name];
    }

    // If name is not already cached, it will be added unless it's value is the same as disallowed
    cache[name] = name !== disallowed;
    console.log(cache);
    return cache;
  }
  return n;
})();

// Call the function to see the results in the console
namesList('Karen'); // will return true
namesList('Jonathan'); // will return false
namesList('Erica'); // will return true
namesList('Karen'); // will return cached data + true
